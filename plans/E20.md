# Plan — Epic E20: Getaway rework & audio that works

**Approach.** Two coherent halves, each landing in dependency order. **Half A — the
Getaway finale (E20.1–E20.2):** make the engine implement design-doc v0.9 §4 — *ditch drops
banked Loot, not Heat; buy-seconds is removed; power-up skips remain* — then redraw the screen
to `8 - Getaway.html` (hero clock, four-column round bar with a **Skips left** column, ARMED →
ACTIVE → near-bust, the Cleared · Skip · Ditch row). **Half B — audio that actually makes sound
(E20.3–E20.5):** the whole audio chain (engine, AudioProvider, soundboard, ambient-bed Heat
nudge) is already wired and tested — the *only* reason nothing plays is that the nine bundled
assets under `src/platform/audio/sounds/*.wav` are 44-byte empty stubs and the manifest covers
only a fraction of the cues the design calls for. So Half B: (E20.3) generate **real, audible,
self-authored CC0** `.wav` assets for the full cue set and expand the manifest to match; (E20.4)
populate the reserved action-bar cue row with phase-contextual quick-cues, surface a clear
*unavailable* state when an asset fails to load, and verify the full board + Heat-driven ambient
bed; (E20.5) make the Getaway itself audible (start cue, ticking that tightens with the clock,
win/bust sting on resolve).

**Architecture note for builders.** The epic text says assets live in `public/sound/`, but the
*current* code resolves them from `src/platform/audio/sounds/*.wav` via a Vite `import.meta.glob`
(`src/platform/audio/soundAssets.ts`) so they inline as data-URIs in the offline build. Follow the
**code**, not the epic prose: replace the stub WAVs in `src/platform/audio/sounds/`, keep the
`.wav` extension (so the existing glob and `soundAssets.test.ts` keep working — no `public/` dir),
and keep the manifest's logical `sound/<file>.wav` `src` paths.

**Asset sourcing decision (no human block needed).** A builder agent cannot reliably reach
freesound.org / a CDN at build time, and the golden rules forbid runtime CDN anyway. The
acceptance gate only requires *audible sound from bundled, openly-licensed assets, licences
recorded*. **Self-authored, procedurally-synthesised audio is CC0 by definition and fully
satisfies that gate.** E20.3 therefore commits a small deterministic generator script that
synthesises each cue as a real PCM WAV (oscillator + noise + envelopes), plus a credits note
marking the assets self-authored CC0. (A builder with genuine network access *may* substitute
real CC0 clips, but must not depend on one.)

test that references `ditchHeatCost`/`buySecondsBonus`).
**Do:**
- Rename the tuning field `getaway.ditchHeatCost` → `getaway.ditchLootCost` and update its `_meta`
  units note (Loot forfeited per ditch). Give it a sensible default consistent with v0.9 economy
  magnitudes (banked Loot is now five-figure; pick a value the design reviewer will read as "some
  banked Loot", e.g. on the order of a few thousand — builder sets a concrete number and notes the
  rationale; it is preset data, not hardcoded in the engine).
- **Remove** `getaway.buySecondsBonus` entirely from the tuning schema (`tuning.ts`), the preset
  JSON, the `EngineConfig` type + `build-config.ts` mapping, and `config.ts`.
- Change the `GETAWAY_DITCH` reducer case (`src/engine/reduce.ts:345-348`) so it **subtracts
  `cfg.getaway.ditchLootCost` from `state.loot`, clamped at ≥ 0, and leaves `state.heat`

  `{ t: 'GETAWAY_DITCH' }` stays the same.
- Fix the now-stale comment in `getaway.ts` (the "subsequent GETAWAY_DITCH events raise Heat in
  state" note) to reflect the Loot-drop rule.
- Update every unit test that asserted ditch raised Heat or that referenced `buySecondsBonus`.
**Acceptance:** dispatching `GETAWAY_DITCH` drops Loot by `ditchLootCost` (never below 0) and

  cues, short one-shots for SFX/stings). No external deps, no network; running it reproduces the
  committed bytes. Keep clips short (≈0.3–2 s) so the offline data-URI build stays small.

  `src/platform/audio/sounds/`: ambient **drone** + **heartbeat**, **ticking clock**, **lock
  click**, **footsteps**, **wire snip**, stings **clean / complication / botch**, **cha-ching**
  (loot), **gear chime**, **alarm/siren**, finale **engine** + **tyres**, **win** sting, **bust**

- Extend `soundAssets.test.ts` to assert every manifest cue resolves to a bundled asset **and**

**Acceptance:** every cue in the expanded manifest resolves to a real, non-empty bundled `.wav`;
