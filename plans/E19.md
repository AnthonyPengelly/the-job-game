# Plan ‚Äî Epic E19: Room fidelity ‚Äî Obstacle commit, Scenario roll reveal, Spoils + cockpit polish

This epic is mostly presentation/IA fidelity against `design-system/redesign/3 - Obstacle Room.html`,
`4 - Scenario Room.html`, `6 - Spoils.html` and `0 - Cockpit Shell.html`, plus one contained pure-engine
bug-fix to the scenario roll's result reporting. The reward *data* (Loot/Gear fields on obstacle options,
gear sell-value, the badged Gear launcher re-assign dialog) already landed in E13/E14 ‚Äî this epic renders
and commits it faithfully. Nothing here invents new design: every visual decision traces to one of the four
redesign HTML files.

Approach: build the shared shell pieces first (the action-bar slot that fixes the "bar covers the stage"
bug for every screen at once, and the redesigned crew cards + commit/attempter rail states), then the three
room screens that consume them, then the scenario engine fix that the dramatic roll reveal depends on, then
Spoils. The current code already has: `CrewRail` tap-to-commit wiring (`crewRailMode.tsx`), an empty grid
`CockpitActionBar`, a `Spoils` screen with tap-card‚Üítap-crew assign + sell button, and the ToolRail badged
Gear re-assign dialog (the `6d` no-dead-end flow). So several acceptance points are partly met and the tasks
below target only what is genuinely missing or wrong.

Golden rules honoured: the engine stays pure and deterministic (the roll fix only re-shapes when the existing
RNG draw is *revealed*, not how it is drawn); all tunables stay preset data; every new state remains
GM-editable / undoable (no dead-ends); the GM-only difficulty dial and roll telemetry never leak to the
player-view surface.
