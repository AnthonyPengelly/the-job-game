# Plan — Epic E17: The coherent heist script (narration + Briefing)

**Approach.** Today the teleprompter picks an *independent* line per beat with no continuity, and the
"Next" button re-rolls a brand-new line on every press (`director.next(beat, ctx)` in
`src/console/teleprompter/director.ts`, called by every screen). E17 turns this into **one coherent
heist per mark**: at run start the narration system commits a **per-mark spine** (mark name, dressing,
vault, stakes) seeded off the run RNG, threads it through every beat via **templating**
(`{mark}`, `{vault}`, `{lane}`, `{crew}`, `{outcome}`, `{heatBand}`, `{runTotal}`…), gives each room an
**entry scene-set** beat and a **reveal/payoff** beat, and redefines **Next** to step through the
*scripted lines of the current beat* (never regenerate). The Briefing loses its design-rejected Order of
Play / Mastermind panel and gains the dossier + "every room pays out" framing per `2 - Briefing.html`.

The work is sequenced so the engine stays pure (golden rule 2 — the spine/templating live in the
content + console layers, never in `src/engine`), content/tuning stays data (golden rule 3 — the spine
and lines are validated presets), and output stays deterministic under seed (golden rule 4 — the
director owns one RNG stream seeded off `runSeed`, and the spine is committed from it). E17.1 lands the
data shapes (schema + initial content + docs); E17.2 lands the pure director/selector logic behind a new
`script()` API *additively* so the UI stays green; E17.3 flips the Teleprompter + all screens onto the
new within-beat "Next" and entry→tension→payoff wiring; E17.4 rebuilds the Briefing; E17.5 is the bulk
content-authoring pass plus the full-run no-repeat/determinism sensor and the design-review gate.

## Task E17.1 — Spine + templating + scene/reveal beat schema and seed content
**Files:**
- `src/content/schema/spine.ts` (new) — Zod `spineSchema` + `markSpineSchema` + `SpineBank`/`MarkSpine` types
- `src/content/schema/narration.ts` — add the new beats; add template-token validation to `narrationVariantSchema`
- `src/content/schema/index.ts`, `src/content/index.ts` — export the new schema/types
- `src/content/narration/template.ts` (new) — pure `fillTemplate(text, ctx)` + the allowed-token list
- `presets/default/content/spine.json` (new) — initial spine bank (≥3 marks per mansionType: villa/estate/penthouse)
- `presets/test-malformed-spine/...` (new, mirror of an existing malformed preset) — a deliberately-broken spine fixture
- `src/platform/presets/browser.ts`, `src/platform/presets/load.ts`, `src/platform/index.ts` — `loadDefaultSpine()` / `loadSpine(id)` mirroring the narration loaders
- `docs/CONTENT-AND-TUNING.md` (and a short note in `docs/MINIGAMES.md`/design pointer if needed) — document the spine schema + the templating token set, as an extension of the E8 narration model
- co-located tests: `src/content/schema/spine.test.ts`, `src/content/narration/template.test.ts`

**Do:**
- Define a **spine** content shape: a `MarkSpine` carries structured dossier fields used both as
  template fill values and as Briefing dossier copy — at minimum `markName` (e.g. "Ashcombe House"),
  `vault` (e.g. "Floor 3"), `security` (e.g. "HIGH"), `targetHaul` (flavour string, e.g. "$120k"),
  `dropCaption` (the image-strip caption), and the opening dressing — keyed/filterable by
  `mansionType` so the committed spine matches the engine-chosen mansion. The bank holds ≥3 marks per
  mansion type. Validate with Zod (`.strict()`, unique ids), parse-at-boundary.
- Add the **new narration beats** to `narrationSchema`: `roomApproach` (scene-set as the crew is about
  to enter — obstacle *and* scenario) and `scenarioReveal` (the scenario payoff on reveal, the scenario
  counterpart of the existing `outcomeQuip`). Keep all existing beats. (Obstacle keeps
  `obstacleClue` as tension and `outcomeQuip` as payoff; scenario uses `scenarioSetup` as tension.)
- Add **templating**: a pure, framework-free `fillTemplate(text, ctx)` that replaces `{token}`
  placeholders from a render context. Define the canonical token set — `{mark}`, `{vault}`,
  `{security}`, `{targetHaul}`, `{lane}`, `{crew}` (committed names joined), `{attempter}`,
  `{outcome}`, `{heatBand}`, `{runTotal}`, `{roomNum}`. A missing context value renders to a safe
  empty/neutral string (never the literal `{token}` at the table). In `narrationVariantSchema`,
  **validate that every `{token}` used is in the allowed set** — unknown tokens fail loudly at parse
  (golden rule 7). No React, no engine imports in `template.ts`.
- Author an initial templated `spine.json` and seed a handful of templated lines so the schema/token
  validators have real coverage; the heavy authoring is E17.5.
- Document the spine schema and token list in `docs/CONTENT-AND-TUNING.md` (it extends the E8 model).

**Acceptance:**
- `spineSchema`/`markSpineSchema` parse the default `spine.json`; malformed spine fixture is rejected.
- `narrationSchema` accepts the two new beats and **rejects** a variant whose text uses an unknown
  `{token}`; existing `narration.json` still parses.
- `fillTemplate` unit tests cover every token, joined `{crew}`, and missing-value → empty behaviour.
- `loadDefaultSpine`/`loadSpine` return a validated bank; `npm run check:full` passes.
**Depends:** none (E13 merged)

## Task E17.2 — Director rework: committed per-beat scripts, spine commit, templating (logic only)
**Files:**
- `src/console/teleprompter/director.ts` — new committed-script API + spine commit + template fill
- `src/console/teleprompter/index.ts` — export new types (`MarkSpine`, script/context types)
- `src/console/teleprompter/director.test.ts` — extend
- (no screen/component changes here — `next()` stays available so the UI stays green)

**Do:**
- Change `createNarrationDirector` to also receive the **spine bank + the committed mansion type**
  (signature/opts change), commit **exactly one** `MarkSpine` for the run by drawing from its own
  RNG **before** any beat selection so the stream stays stable and deterministic, and expose it
  (e.g. `director.spine`). Selection is filtered to the engine-chosen `mansionType` and is reproducible
  from `runSeed`.
- Add a **committed-script** method — `director.script(beat, ctx): string[]` — that returns the ordered
  list of scripted lines for *this occurrence* of the beat: it picks the line(s) once (advancing the
  beat's recent-avoiding ring buffer from E8 so subsequent occurrences differ and a full run does not
  repeat), and returns one-or-more **template-filled** lines (spine + the supplied `ctx` merged). A
  beat occurrence is committed once and is stable for that occurrence — calling it again for the same
  occurrence is the screen's responsibility to avoid (screens hold the array). Keep determinism: same
  seed + same call order + same ctx ⇒ same scripts.
- Thread `fillTemplate` so returned lines already have `{mark}`/`{vault}`/`{lane}`/… resolved from the
  spine and `ctx`.
- Keep the legacy `next()` method working (delegating to the new internals) so E17.1/E17.2 land without
  touching screens — the UI migration is E17.3.

**Acceptance:**
- `director.script(beat, ctx)` is deterministic under seed and returns template-filled lines; spine is
  committed once per run and matches `mansionType`.
- No-repeat property test: across a full run's worth of `script()` calls per beat, lines do not repeat
  until the beat pool is exhausted (E8 selector property preserved).
- Existing director tests still pass (legacy `next()` intact); `npm run check:full` passes.
**Depends:** E17.1

## Task E17.3 — Teleprompter "Next within beat" + rewire all screens to scripts
**Files:**
- `src/console/teleprompter/Teleprompter.tsx` + `Teleprompter.test.tsx` — step through a committed line array; show **Next** only when a next line exists
- `src/console/store/store.ts`, `src/console/store/StoreProvider.tsx` — pass the spine bank + committed mansion type into `makeDirector` on startRun/hydrate/goAgain/applyPreset
- `src/console/screens/ObstacleRoom.tsx`, `ScenarioRoom.tsx`, `Spoils.tsx`, `Offer.tsx`, `Getaway.tsx`, `Result.tsx` — migrate from `director.next()` to `director.script()`; wire entry→tension→payoff
- drop the legacy `director.next()` once no caller remains; update affected screen tests

**Do:**
- Rework the `Teleprompter` presenter so it is handed the **committed line array** (or a current
  line + a `hasNext` flag) and an `onAdvance` that steps to the next line *within the beat*. The Next
  button **only renders when there is a next line** and never regenerates/re-rolls (epic "Next"
  semantics). No disabled dead-end — when there is no next line, the control is simply absent.
- In each screen, replace the `next()`-on-mount + `next()`-on-advance pattern with a single
  `script(beat, ctx)` call committed once (e.g. `useState` initializer / single-fire effect as the
  screen already does), holding the returned array and a local index that `onAdvance` increments.
- Wire **entry → tension → payoff** per room:
  - Obstacle: `roomApproach` (scene-set on entry, before the option cards) → `obstacleClue`
    (the choice ahead) → `outcomeQuip` in Spoils (payoff, existing).
  - Scenario: `roomApproach` (entry) → `scenarioSetup` (the choice ahead, existing) →
    `scenarioReveal` in Spoils for scenario rooms (payoff — extend Spoils, which currently shows a quip
    for obstacle rooms only).
  - Build each screen's `ctx` from live state (lane, committed crew names, attempter, outcome,
    heatBand, run total/loot, room number) so templated tokens resolve.
- Update `store.ts`/`StoreProvider.tsx` so `makeDirector` receives the spine bank and the
  freshly-committed `mansion.type` (available on the reduced/replayed session) at every director
  (re)creation; the pre-run/seed-0 director may use the default mansion.
- Remove `next()` after migration; keep all GM-override/no-dead-end behaviour intact.

**Acceptance:**
- Pressing **Next** advances through the current beat's scripted lines and disappears at the last line;
  it never produces a freshly-rolled line.
- Each room is scene-set on entry and paid off on reveal; scenario rooms now get a reveal line.
- Templated tokens render real values (no literal `{...}` on screen); player-view isolation untouched.
- Screen/Teleprompter tests updated and green; `npm run check:full` passes.
**Depends:** E17.2
