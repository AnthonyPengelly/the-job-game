# Plan — Epic E18: Mini-game fidelity, full-team games & judging fixes

**Approach.** E13 themed the run shell but left the ten mini-game *panels* essentially undrawn versus their
`5b`/`5c` mockups, the combo games still carry the old **two-boost-per-game** model, no game can take the whole
table, and `Categories` (plus siblings) downgrade a met target to a complication when the timer expires. This
plan lands the three cross-cutting *logic/data* fixes first — the boost-model collapse, the full-team capability,
and the judging fix — so the panels are redrawn on top of a settled contract, then redraws each of the ten games
(and the solo/negotiated variants) to its specific `5b`/`5c` panel in three grouped tasks.

The boost-model collapse is deliberately a **single** task even though it spans every combo game: it is one
logical change (one ability per game, fire on any relevant lane) that must keep the repo green — leaving some
games with two boosts while the shared UI assumes one would break `check:full` between tasks. The redraw tasks
that follow touch only component visuals/instructions, not boost or judge logic, so they don't re-litigate it.

**Authority note — Steady Breath.** The epic's acceptance-gate prose says "boosts read as their lane power-up
*(Steady Breath retained)*", which directly contradicts the same epic's story body ("**Steady Breath** … are
retired") **and** design doc **v0.9 §5** ("*Steady Breath … are retired*"). The story body + the v0.9 design doc
are the higher authority (the gate parenthetical is a typo), so this plan **retires Steady Breath** along with the
other five dropped boosts. This is resolved by the docs themselves; it is **not** a blocking question.

**Authority note — boosts follow v0.9, not the mockups.** The `5b`/`5c` mockups and `FRONTEND-REDESIGN-BRIEF.md`
predate the boost fix and still show two boosts per combo game. Per the epic, follow design doc **v0.9** (one
ability per game) for *boosts only*; follow the mockups for everything else (layout, panels, controls).

**Mockup sources:** `design-system/redesign/5 - Mini-games.html`, `5b - The Ten Games 1-5.html`,
`5c - The Ten Games 6-10.html`, and `docs/FRONTEND-REDESIGN-BRIEF.md`.

**Full-team decision (documented here, per the epic's "decide and document"):** the three "confer as a table"
games are full-team — **Categories, Inside Knowledge, Assembly Line** (incl. the negotiated variant). These match
design doc §"the ten games" (crew rattles off / crew confers / whole table trades) and have no meaningful crew
subset. All other games keep commit + rotation.

- Single-lane games stay one ability but confirm labels: Crack the Tumblers=**Reset Pin**, Beat 16=**In the Bones**, Categories=**Skip**, The Once-Over=**Hunch** (`crack-the-tumblers/*`, `crack-the-tumblers-solo/*`, `beat-16/*`, `categories/*`, `the-once-over/*`)

**Do:** Implement the v0.9 boost model. `boosts` carries **exactly one** `BoostHook` per game, and a committed
player may fire it if they hold a power-up in **any** lane in the game's `lanes` (not the boost's single lane).
Update `BoostButton`/`MinigameHost` eligibility to test `game.lanes.some(l => player.powerUps[l])` rather than
`powerUps[hook.lane]`. Collapse every combo game's dual boost to its single signature ability and **delete** the
six retired boosts' forgiveness logic from judges/components/content/tests. Ensure the ARMED preview and the
ACTIVE in-game boost control render as one shout with **no layout shift** when it fires (reserve space). Keep the
rest of the `MiniGame` contract stable. If the contract *shape* (not just data) must change to express
"one ability, any relevant lane", make that change here — this is the one task sanctioned to touch the boost model.

**Acceptance:** every game exposes exactly one boost whose label matches the v0.9 list above; eligibility fires on
a power-up in any of the game's lanes (covered by a `BoostButton`/host test for a combo game with only the *other*
lane's power-up held); no occurrence of the six retired boost names remains in `src/`, `presets/`, or tests
(`grep` clean); `docs/MINIGAMES.md` boost section reflects one-ability-per-game; `npm run check:full` passes.
**Depends:** E13 (merged)

## Task E18.2 — Full-team capability: contract flag, no crew-select, no resting
**Files:**
- `src/minigames/contract.ts` (add `fullTeam?: boolean` to `MiniGame`)
- `src/minigames/games/categories/index.ts`, `inside-knowledge/index.ts`, `assembly-line/index.ts`, `assembly-line-negotiated/index.ts` (set `fullTeam: true`)
- `src/console/screens/MinigameHost.tsx` + `src/console/shell/crewRailMode.tsx` + `src/console/screens/ObstacleRoom.tsx` (when the chosen option's game is `fullTeam`, commit the **whole crew** and skip the crew-select step)
- `src/engine/reduce.ts`, `src/engine/crew.ts`, `src/engine/types.ts` (no exhaustion/`restingUntilRoom` applied for a full-team game's committed crew)
- `docs/MINIGAMES.md` (document the three full-team games and the no-select/no-rest rule), scaling/preset docs in `presets/default/` if commit-range or rotation tuning is encoded there
- Tests: engine reducer test (full-team resolution applies **no** resting/rotation) + a host/crew-rail test (full-team game shows no crew-select)

**Do:** Add a `fullTeam` capability. For a game flagged `fullTeam`, the host takes the entire current (non-resting?
— see note) crew automatically, renders **no** crew-select UI, and the engine applies **no** exhaustion so those
players are **not** rested next room. Non-full-team games keep the existing commit + rotation untouched. Mark
Categories, Inside Knowledge, Assembly Line (+ negotiated) as full-team per the decision documented above. Keep
GM override intact — the GM can still edit committed crew/outcome (golden rule: no dead-ends).

**Note for builder:** "whole crew" means every player at the table for that game; because full-team games impose
no resting, exhaustion rotation simply doesn't apply to them. If the mockups/`docs/MINIGAMES.md` imply benched
players still don't play their *own* commit elsewhere, keep it simple: full-team = everyone participates, nobody
is rested afterwards. Do not invent extra rotation rules.

**Acceptance:** the three designated games run with no crew-select and leave no player resting the next room
(engine-tested); all other games keep commit + rotation; `docs/MINIGAMES.md` lists the full-team games; the
`fullTeam` flag is the only contract addition; `npm run check:full` passes.
**Depends:** E18.1
